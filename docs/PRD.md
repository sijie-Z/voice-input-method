# 语音输入法 — 产品设计文档（PRD）

## 产品概述

一款**离线优先、AI 增强**的 Windows 语音输入工具。按住右 Ctrl 说话，松开自动识别、润色并上屏到任意应用，支持 16 条语音指令操控系统。

## 目标用户

- 需要大量文字输入但打字速度慢的办公人群
- 双手需要同时操作但需要记录想法的创作者
- 会议中需要快速做纪要的职场人士
- 对隐私敏感、不信任云端语音服务的用户

## 用户痛点

| 痛点 | 现有方案 | 我们的解法 |
|------|----------|-----------|
| 语音输入需联网，隐私泄露 | 讯飞/搜狗上传云端 | Vosk 离线识别，音频不离开本机 |
| 口语化表达不流畅 | 输出原文需手动修改 | DeepSeek AI 润色，自动去口语化 |
| 语音与键盘操作割裂 | 说完整句再手动编辑 | 语音指令直控系统（换行/删除/搜索） |
| 不同场景表达风格不一 | 手动调整措辞 | 四种润色模式（通用/会议/邮件/朋友圈） |
| 修改后内容需要重复粘贴 | 复制粘贴反复切换 | 自动上屏到当前窗口 |

## 核心功能

| 功能 | 优先级 | 解决的问题 |
|------|--------|-----------|
| 按住右 Ctrl 录音，松开上屏 | P0 | 核心输入闭环，零学习成本 |
| Vosk 离线中文语音识别 | P0 | 隐私安全，无网络可用 |
| AI 文本润色（4 种模式） | P1 | 口语转书面语，适配不同场景 |
| 16 条语音系统指令 | P1 | 语音直控编辑器、浏览器、系统 |
| 剪贴板保护 | P2 | 不影响用户原有复制内容 |
| 指令热词增强 | P2 | 提升指令识别命中率 |

## 技术选型理由

| 决策 | 方案 | 理由 |
|------|------|------|
| 语音识别引擎 | Vosk（离线） vs Web Speech API | Vosk 支持热词、不依赖浏览器、无网络需求、识别质量稳定 |
| AI 润色 | DeepSeek 通过 OpenAI SDK | API 成本低、中文能力强、兼容 OpenAI 格式便于切换 |
| 音频采集 | sounddevice + 回调模式 | 非阻塞录音、低延迟、线程安全 |
| 模拟输入 | pyautogui + pyperclip | 跨窗口粘贴，兼容所有应用 |
| 全局热键 | pynput 右 Ctrl | 不干扰系统快捷键，单手操作 |

## 竞品分析

| 维度 | 讯飞输入法 | 搜狗输入法 | 本产品 |
|------|-----------|-----------|--------|
| 离线识别 | 部分支持 | 依赖云端 | 完全离线 |
| 语音指令 | 无 | 无 | 16 条系统级指令 |
| AI 润色 | 无 | 无 | 4 种润色模式 |
| 上屏方式 | 输入法框架嵌入 | 输入法框架嵌入 | 剪贴板粘贴，兼容所有应用 |
| 隐私 | 音频上传云端 | 音频上传云端 | 零上传 |
| 开源性 | 闭源 | 闭源 | MIT 开源 |

## 未来规划

- **自定义热词配置**：支持 JSON 文件配置热词，无需改代码
- **连续对话模式**：长按期间实时显示识别中间结果
- **多语言支持**：英文、日文离线模型切换
- **音色识别**：区分不同说话人，适用于会议场景
- **Tray 托盘驻留**：后台运行不占任务栏空间
